# 🚀 Quiz Platform & Telegram Mini-App – Обзор Проекта

Этот документ подготовлен для использования в вашем портфолио. Он детально описывает архитектуру системы, бизнес-фичи и технические интеграции проекта.

## 📋 О проекте: Бизнес-ценность и Продукт

Этот проект — не просто Telegram-бот, а **полноценное масштабируемое SaaS-решение (Software as a Service)**, разработанное для быстрого запуска и монетизации образовательных продуктов. Платформа позволяет создавать неограниченное количество полностью независимых проектов на различных языках и тематиках на основе единого "ядра" и общей базы управления.

**В чем моя ценность как инженера для бизнеса:**
*   **Time-to-Market (Доля рынка и скорость вывода продуктов):** Я спроектировал архитектуру так, что запуск нового нишевого продукта (с отдельным брендом, Telegram-ботом, доменом и Mini App) занимает **всего несколько кликов и минут работы** в панели администратора, без необходимости писать новый код или нанимать команды программистов. Это кардинально экономит бюджет бизнеса на запуск и тестирование гипотез.
*   **Высокая конверсия и Удержание (Retention):** Я создал бесшовный пользовательский опыт, интегрировав платформу прямо внутрь Telegram (в виде нативного мобильного приложения / Mini App) с механикой свайпов (как в Tinder). Пользователи могут учиться и проходить квизы, не покидая мессенджер, что резко повышает вовлеченность и время, проводимое в приложении.
*   **Глобальная инфраструктура платежей:** Я интегрировал прием платежей, работающий по всему миру. Поддерживаются банковские карты (Stripe), внутренняя валюта экосистемы Telegram (Telegram Stars ⭐️) и криптовалютные шлюзы (Wallet Pay / CoinGate). Это позволяет бизнесу продавать премиум-подписки клиентам из любой точки мира без дополнительных интеграций.
*   **Снижение издержек на SMM:** Я реализовал автоматизированную публикацию контента (quiz задач) во все популярные социальные сети (Pinterest, Дзен, Instagram, Facebook, Shorts). Это генерирует бесплатный органический трафик на платформу без участия контент-менеджеров.

## 🏗 Архитектура Системы

Проект использует современную архитектуру, состоящую из следующих компонентов:

1.  **Backend (Monolith API & Admin Panel):**
    *   **Фреймворк:** Django & Django REST Framework (DRF)
    *   **База данных:** PostgreSQL 15+ с изоляцией данных на уровне записей (Row-Level Security) через общий `tenant_id`. Эта архитектура (Shared Database, Shared Schema) упрощает миграции, обеспечивая полную безопасность данных между проектами. 
    *   **Фоновые задачи:** Celery & Redis для асинхронных процессов (публикация постов, отправка писем, генерация отчетов, обработка медиа).
    *   **Функции:** Управление контентом, изоляция пользователей, выставление REST API для Mini App.

2.  **Telegram Mini App:**
    *   **Фреймворк:** FastAPI (асинхронный и высокопроизводительный)
    *   **Особенности:** Кастомный фронтенд на HTML/CSS/JS (Vanilla JS для максимальной производительности, отсутствие тяжелых фреймворков для мгновенной загрузки в Telegram).
    *   **Роутинг и API:** Обмен данными с Django API, кэширование (Redis). 

3.  **Telegram Bot:**
    *   **Фреймворк:** `aiogram` (v3) 
    *   Сборщик событий, инлайн-квизы, уведомления и прием платежей через интегрированные системы.

4.  **Инфраструктура и DevOps:**
    *   Контейнеризация через **Docker & Docker Compose**.
    *   Web-сервер **Nginx** как реверс-прокси с автоматической генерацией SSL-сертификатов (Certbot/Let's Encrypt).
    *   **Cloudflare R2** как S3-совместимое хранилище (CDN) для быстрой отдачи медиа-файлов, видео и изображений (достигнута высокая производительность LCP/FCP). 
    *   **CI/CD & Скрипты развертывания:** Множество написанных bash-скриптов для автоматического бекапа БД, деплоя, проверки работы Nginx и SSL.

## 🌟 Основные Бизнес-Функции

*   **Multi-Tenancy (SaaS):**
    Уникальный `TenantMiddleware` динамически определяет контекст (`domain` или `hostname`) и отдает нужные данные, дизайн (seo, брендинг) и Telegram ботов. Администраторы разделены по инстансам (`TenantFilteredAdminMixin`), но Superuser видит все. Запуск нового Telegram бота под независимый бренд достигается в пару кликов.
*   **Геймификация и Квизы:**
    Разноуровневые квизы (Задачи, Темы, Подтемы), динамическая статистика с отслеживанием прогресса. Реализован уникальный UI в стиле свайп-карточек (Tinder-like swipe) в Mini App для выбора вопросов.
*   **Авто-генерация Видео и Медиа-парсинг:** 
    Интеграция с автоматической генерацией шортс/рилс-видео с вопросами на основе текста, а также система кэширования превью-изображений.
*   **Локализация (i18n):** 
    Полная поддержка мультиязычности (RU, EN). Связывание переводов на уровне БД с использованием `translation_group_id`.
*   **Мощный движок Уведомлений:**
    Умная система уведомлений (`Notification` + Telegram API). Пользователи получают обновления при ответах в комментариях, админы — пуш-уведомления через бота при новых жалобах, подписках или подозрительной активности.

## 💰 Системы Монетизации & Платежи

Проект поддерживает множество фиатных и крипто-платежных решений:

1.  **Telegram Stars (⭐️):** 
    Нативная интеграция Telegram XTR валюты с `Webapp.openInvoice()`. Реализована полная отчетность в Django админке по каждому платежу Stars с красивыми виджетами.
2.  **Stripe Integration:** 
    Глобальные платежи (USD, EUR, RUB). Обработка всех состояний платежа по Webhook от Stripe с fallback-защитой.
3.  **Telegram Wallet Pay & CoinGate:**
    Оплата стейблкоинами (USDT, TON, BTC) двумя путями: нативным Telegram кошельком (Wallet API) для простоты и CoinGate для веб-пользователей.

## 📱 Дизайн и Производительность
*   Полная **SEO-оптимизация** на Django-фронтенде: генерация стэмп-мета-тегов JSON-LD, динамическая карта сайта (Sitemap), контроль `canonical` ссылок (исключены проблемы дублирования между тенантами) и OpenGraph. 
*   **Cache Busting** (`cache_buster.py`): Автоматическое добавление версий к статике, чтобы обходить кэш браузеров/Telegram при релизах. Оптимизация First Contentful Paint.
*   Интерактивные объяснения с продвинутым парсером `source_name` и `source_link` Markdown формата прямо в модальных компонентах. 

## 🌐 Автоматическая публикация SMM (Social Media)
Разработана система интеграции (через Django Signals и Make.com вебхуки):
*   Публикация заданий напрямую в **Pinterest** API, **Facebook** Page API.
*   Интеграция через Webhooks с Яндекс.Дзен, Instagram, TikTok и YouTube Shorts.
*   Мощная система логирования `SocialMediaPost`, отслеживание статуса `pending`, `published`, `failed` с возможностью перевызова `retry`.

## 🛠 Технический Стек (Summary)

*   **Databases / Cache:** PostgreSQL 15, Redis
*   **DevOps / Cloud:** Docker & docker-compose, Nginx, Облачное S3 (Cloudflare R2), Let's Encrypt
*   **Frontend:** HTML5, CSS3 (Custom Design System without Tailwind to ensure ultra-low bundle size), JS (Vanilla, DOM manipulation API)
*   **Third-party:** Telegram Bot API, Telegram Stars, Stripe, Wallet Pay, CoinGate, Make.com
