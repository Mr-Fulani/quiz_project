# Документация: Источники для объяснений задач (Explanation Sources)

## Описание функции
В рамках улучшения платформы была добавлена система продвинутого отображения внешних источников для объяснений квиз-задач. Администраторы теперь могут указывать, откуда была взята информация (например, ссылки на документацию, Википедию, хадисы и т.д.), а фронтенд красиво и автоматически преобразует эти ссылки в кликабельные элементы.

## Как это реализовано технически
1. **База данных (`TaskTranslation`):** 
   - Добавлены два дополнительных поля: `source_name` (название источника) и `source_link` (прямая ссылка). 
2. **Админ-панель (`tasks/admin.py`):**
   - Блок редактирования переводов задачи изменен на `StackedInline` для удобного ввода новой информации (старый `TabularInline` был неудобен при наличии большого числа текстовых полей).
3. **Фронтенд (`quiz_combined.js`):**
   - Добавлена продвинутая функция `formatInlineLinks`, которая "на лету" анализирует текст из базы данных и автоматически находит как **прямые ссылки** (начинающиеся на http/https), так и **Markdown-ссылки**.
   - Эта обработка применяется как к полю "Источник", так и к основному телу подробного объяснения (`long_explanation`).

## Как добавлять источники (Инструкция для редакторов)

Указывать источники можно двумя путями:

### Способ 1: Одиночный четкий источник
Если у объяснения есть один конкретный источник:
1. Заполните поле **"Название источника"** (например, *Википедия: Алгоритм Дейкстры*)
2. Заполните поле **"Ссылка на источник"** (например, *https://ru.wikipedia.org/...*)

На фронтенде это сформирует четкую кликабельную ссылку с именем автора/источника в самом низу объяснения.

### Способ 2: Множественные источники (Продвинутый режим)
Если источников несколько, поле "Ссылка на источник" можно **оставить полностью пустым**.
Затем в поле **"Название источника"** впишите все свои источники одним текстом. Система поддерживает два варианта автораспознавания ссылок:

**А) "Сырые" ссылки рядом с текстом:**
Вы можете просто вставлять ссылки после текста. Если после ссылки стоит знак препинания (запятая, точка, точка с запятой), алгоритм аккуратно "отрежет" его от самой ссылки, чтобы она не сломалась.
*Пример ввода:*
> `Sahih al-Bukhari, Book 2, Hadith 8 https://sunnah.com/bukhari/2/8; Sahih Muslim, Book 1, Hadith 16 https://sunnah.com/muslim/1/16`

**Б) Красивые Markdown-ссылки:**
Для максимальной эстетики вы можете использовать Markdown: `[Текст, который видит пользователь](Сама ссылка)`.
*Пример ввода:*
> `[Sahih al-Bukhari, Book 2, Hadith 8](https://sunnah.com/bukhari/2/8); [Sahih Muslim, Book 1, Hadith 16](https://sunnah.com/muslim/1/16)`

## Форматирование ссылок в самом тексте объяснений
Этот же алгоритм (`formatInlineLinks`) теперь автоматически применяется и ко **всему длинному тексту объяснения** задачи (`long_explanation`). Если редактор вставляет ссылки или Markdown-ссылки прямо посреди параграфов с объяснением, система автоматически зарендерит из них кликабельные голубые ссылки, выделяющиеся на фоне текста. Это снимает с редакторов задачу вручную прописывать HTML теги `<a>`.
